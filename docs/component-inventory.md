## Overview

The component library follows a **two-tier architecture**:

1. **UI Components** (`app/components/ui/`) — Base primitives inspired by shadcn/ui
2. **Layout Components** (`app/components/layout/`) — Page section compositions

All components follow consistent patterns:

- TypeScript interfaces with explicit props
- `forwardRef` for DOM access where needed
- `cn()` utility (clsx + tailwind-merge) for class merging
- Storybook stories with Chromatic visual regression
- WCAG 2.1 AA accessibility compliance
- Pregnancy-safe design (touch targets ≥44px, reduced motion support)

| **Composition**                    | Pages compose layout sections which compose UI primitives |
